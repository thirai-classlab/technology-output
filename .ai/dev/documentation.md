# 設計資料の作成ルール

## 出力先

- 設計資料は `docs/design/` に出力する

## フォーマット

- Markdownで記述する
- 図解が必要な箇所にはMermaidを使用する
- 資料は完成形として出力し、確認事項や質問を資料内に含めない
- 確認・質問・フィードバック依頼はチャットで行う

## 命名

- `{NN}-{内容}.md` 形式
- 連番で管理する

## 確定後の反映

- 設計書のステータスが「確定」になったら、その中のルール・テンプレート・手順を `.ai/` の該当カテゴリに反映する
- `docs/design/` は設計の経緯を残すアーカイブ、`.ai/` がAIが実際に参照する実行ルール
- 反映先の対応:
  - ワークフロー・構造の設計 → `.ai/dev/` 配下
  - テンプレート → `.ai/content/{該当カテゴリ}/`
  - 変換・投稿ルール → `.ai/content/posting/`
- 反映後、CLAUDE.md も必要に応じて更新する
