# Qiita 投稿ルール

## フォーマット対応状況

| 要素 | 対応 | 記法 |
|------|------|------|
| 見出し | H1-H6 | `#` の後に半角スペース必須 |
| 太字 | 対応 | `**text**`（前後にスペース推奨） |
| 斜体 | 対応 | `*text*` |
| 取り消し線 | 対応 | `~~text~~` |
| テーブル | 対応 | GFM形式（配置指定可） |
| コードブロック | 対応 | 200+言語（Rouge）、ファイル名表示可 |
| Diff表示 | 対応 | `diff_python` で言語別Diff表示 |
| Mermaid | 対応 | 2,000字/ブロック、Chain数10以下 |
| 数式 | 対応 | ブロック: ` ```math `、インライン: `` $`式`$ `` |
| 脚注 | 対応 | `[^1]` / `[^1]: 内容` |
| 折りたたみ | 対応 | `<details><summary>` |
| 注釈ブロック | 対応 | `:::note info/warn/alert` |
| 埋め込み | 対応 | URL直貼り（カード化）、iframe |
| 目次 | 自動生成 | 記事右側に自動表示 |
| 絵文字 | 対応 | `:emoji_code:` 形式 |

## 独自記法

### Note記法（注釈ブロック）

master.md の注意書き・補足を以下に変換する:

```markdown
:::note info
補足情報テキスト
:::

:::note warn
注意喚起テキスト
:::

:::note alert
警告テキスト
:::
```

### コードブロック（ファイル名 + Diff）

```markdown
```python:sample.py
print("hello")
```

```diff_python
+ added_line
- removed_line
```
```

### 折りたたみ

長いコード例や補足情報に使用:

```markdown
<details><summary>詳細を見る</summary>

折りたたまれた内容（Markdown記法使用可）

</details>
```

注意: `<summary>` 閉じタグの直後に**空行が必須**。

### 脚注

```markdown
参照テキスト[^1]

[^1]: 脚注の内容
```

## 構成

- 冒頭に「この記事の対象読者」と「この記事で得られること」
- 技術的に正確・具体的な内容
- 末尾に参考リンク
- タグは最大5つ設定（記事内容に合った技術タグ）

## トーン

- 技術記事調（ですます体）
- 主観を入れる場合は根拠を添える
- 再現可能なコードサンプルを心がける

## 推奨

- 文字数: 3,000〜6,000字が最適
- コードブロックには必ず言語指定を付ける
- 画像: WordPressにアップロード済みのURLを参照（単体10MB、月間100MB上限）

## 投稿手段

- qiita-cli（https://github.com/increments/qiita-cli）
- `npx qiita preview` で下書きプレビュー → 確認 → `npx qiita publish` で公開
